基础语法

  1. 变量 var name = value or name := value, string是Go语言内键类型
  2. 常量 const name = value
  3. if - else if - else
  4. 只有for循环
  5. switch语句
    1. 不需要写break
    2. case后可以接任意类型，接 condition 可代替传统的 if-else 条件分支
  6. 数组 不常使用 s : = [len]int
  7. 切片(slice) 可变长度的数组
    1. 使用make创建切片 s := make([]string, len)
    2. 使用append添加 s = append(s, var)
    3. s[num1, num2]
  8. map m := make(map[string]int) 无序
  9. range for var1, var2 := range slice/map
    1. 遍历slice，第一个值返回索引， 第二个值返回值
    2. 遍历map， 返回值对应 key-value
  10. 函数 func func funcName(parameter,...) (returnVal,...){}
  11. 指针 声明 *type 使用 &name
  12. 结构体(struct) 带类型字段的集合 type name struct{} val := name{}
  13. 结构体方法 func (name (*)structName) funcName() {} 结构体方法就是类成员方法，结构体有指针的区别
  14. 错误处理 error errors.New("not fount")
  15. 字符串操作 strings
  16. 字符串格式化 %v匹配任意类型 %+v 字段名和值 %#v 更详细的信息
  17. JSON 处理
    1. 结构体内变量名称首字母大写，就可使用 json.Marshal(structVal) 序列化成byte数组, 再使用string()强转成字符串
    2. `json:"nickname"`写在结构体字段声明后, 可以使打印时, 显示nickname
    3. json.Unmarshal(parameter) 反序列化
  18. 时间处理
    1. t := time.now()
    2. t := time.Date(2022, 5, 7, 10, 0, 0, 0, time.UTC) or t.Format("2022-05-07 10:00")
    3. t.Year(), t.Month(), t.Day(), t.Hour(), t.Minute()
    4. t.Unix() 获取时间戳
  19. 数字解析 strconv.ParseInt("111", 10 , 64) 10表示进制, 64表示进度
  20. 进程信息
    1. os.Args 获取命令行参数
    2. os.Getenv("PATH) 获取环境变量 os.Setenv("PATH",...)
    3. exec.Command("grep", "127.0.0.1", "/etc/hosts").CombinedOutput() 启动子进程

Go语言上手 - 工程实践

并发和并行

  1. Goroutine
    1. 协程: 用户态, 轻量级, 栈kb级别
    2. 线程: 内核态, 线程跑多个协程, 栈mb级别
  2. CSP(Communicating Sequential Processes)
    1. 通过通信共享内存实现通信: 多个goroutine之间通过通道进行通信
    2. 通过共享内存实现通信: 多个goroutine之间在临界区里进行通信
    3. 提倡通过通信共享内存实现通信
  3. Channel make(chan type,[缓冲大小])
    1. 无缓冲通道 make(chan int) 同步通道
    2. 有缓冲通道 make(chan int, 2)
  4. sync
    1. 并发安全 Lock lock sync.Mutex lock.Lock(), lock.Unlock()
    2. WaitGroup 计数器 Add(delta int)+delta Done()-1 Wait() 阻塞到计数器为0

依赖管理

  1. GOPATH
    1. $GOPATH 环境变量
    2. bin 项目编译二进制文件
    3. pkg 项目编译的中间产物, 加速编译
    4. src 项目源码(项目代码直接依赖src下的代码)
    5. go get下载最新版本的包到src目录下
  2. Go Vendor
    1. 项目目录下增加vendor文件, 所有依赖包副本形式放在 $ProjectRoot/vendor
    2. vendor -> $GOPATH
    3. 弊端: 无法控制依赖的版本 更新项目又可能出现依赖冲突
  3. Go Module
    1. 通过go.mod文件管理依赖包版本
    2. 通过go get/go mod 指令工具管理依赖包
    3. 可以定义版本规则和管理项目依赖关系
  4. 依赖管理三要素
    1. 配置文件, 描述依赖 go.mod
    2. 中心仓库管理依赖库 Proxy
    3. 本地工具 go get/mod
  5. 依赖配置 go.mod
    1. module … 依赖管理基本单元
    2. go 版本号 原生库
    3. require () 单元依赖
    4. version ${MAJOR}.${MINOR}.${PATCH} commit伪版本号 indirect 间接依赖 +incompatible
    5. 依赖图
  6. 依赖分发 回源
    1. Github/SVN -> Developer 1.无法保证稳定性 2.无法保证依赖可用性 3.增加第三方压力
    2. Github/SVN -> Proxy -> Developer 稳定可靠
    3. 变量 GOPROXY = “https://proxy1.cn, https://proxy2.cn, direct” direct表示源站
    4. 工具 go mod {init 初始化 download 下载模块到本地缓存 tidy 增加需要的依赖删除不需要的依赖}

单元测试

  1. 规则
    1. 所有测试文件以 _test.go 结尾
    2. func TestXxx(*testing.T)
    3. 初始化逻辑放到 TestMain 中
  2. 运行 go test [flags][packages]
  3. assert 测试包
  4. 覆盖率
  5. Tips
    1. 一般覆盖率 : 50-60 较高覆盖率 80+
    2. 测试分支相互独立, 全面覆盖
    3. 测试单元粒度足够小, 函数单一职责
  6. 文件处理
  7. Mock

高质量编程

高质量编程简介
  1. 编写的代码能够达到正确可靠, 简洁清晰的目标可称之为高质量代码
  2. 要点:
    1. 边界条件是否考虑完备
    2. 异常处理, 稳定性
    3. 稳定易维护
编码规范
  1. 编程原则
    1. 简单性
    2. 可读性 代码是写给别人看的, 不是写给机器看的
    3. 生产力 团队整体工作效率非常重要
  2. 规范
    1. 代码格式
    2. 注释
    3. 命名规范
    4. 控制流程
    5. 错误和异常处理
性能优化建议
  1. slice
    1. 预分配内存
    2. 冗余内存释放
  2. map 预分配内存
  3. 字符串的处理 strings.Builder 和 bytes.Buffer
  4. 多线程的线程安全 使用atomic包
  5. 不要一味地追求程序的性能, 越高级的性能优化手段越容易出现问题
