3).chapter03 变量 Variable

变量定义 Variable

  1. 变量是程序的基本组成单位
  2. 变量相当于内存中的一个数据存储空间的表示，你可以把变量看做是一个房间的们牌号，通过门牌号我们可以找到房间，而通过变量名可以访问到变量
  3. 变量使用的基本步骤 声明->赋值
  4. 不同变量，类型不同，占用的空间大小不同
  5. 变量在同一作用域不能重名

简单了解字符编码

  1. ASCII(一个字节表示，128个字符，实际上1个字节可以表示256个字符，只用了128个)
  2. Unicode(固定大小的编码，使用2个字节来表示字符，字母和汉字统一都是占用2个字节，这样浪费空间),兼容ASCII编码
  3. UTF-8(大小可变的编码，字母使用1个字节，汉字使用3个字节)，是Unicode编码的改进版
  4. GBK(可以表示汉字，而且范围广，字母使用1个字节，汉字使用2个字节)
  5. gb2312(可以表示汉字，gb2312 < GBK)
  6. big5码(繁体中文，台湾，香港)

基本数据类型转换

1.自动类型转换(auto-convert)
  1. 当Java程序在进行赋值或者运算时，精度小的类型自动转换为精度大的数据类型
  2. 数据类型按精度(容量)大小排序

char -> int -> long -> float -> double

byte -> short -> int -> long -> float ->double

  1. 当有多种类型的数据混合运算时，系统首先自动将所有数据转换成容量最大的数据类型，然后再进行计算
  2. char，byte，short 之间不发生自动类型转换，当把具体的数值赋给它们时，先判断该数是否在其范围内,如果使用变量赋值，则判断类型
  3. char，byte，short 在计算时，首先转换成int
  4. boolean 不参与自动转换
  5. 自动提升原则：表达式结果类型自动提升为操作数中最大的类型
2.强制类型转换(force-convert)
  1. 当进行数据从大到小转换时，就需要使用强制类型转换
  2. 需要转换一串运算结果时，常用小括号提升优先级
  3. 强制类型转换会造成精度损失

进制 Into the system

  1. 二进制 binary 八进制 octal 十进制 decimal 十六进制 hex
  2. 二进制显式存储，位数必须为8的倍数，如0b00010101，因为一个字节有8个空间(bit)

1.二，八，十六进制转十进制

每位 * (2,8,16)的(n-1)次方

2.十进制转二，八，十六进制

将十进制数，不断除以(2,8,16)，直到商为0为止，然后将每步得到的余数倒过来，就是对应的二，八，十六进制数

3.二进制转八，十六进制
  1. 二进制转八进制：从低位开始，将二进制数每三位一组，转成对应的八进制数即可
  2. 二进制转十六进制：从低位开始，将二进制每四位一组，转成对应的十六进制即可
4.八，十六进制转二进制
  1. 将八进制数每位，转成对应的一个3位的二进制数即可
  2. 将十六进制数每位，转成对应的一个4位的二进制数即可

原码，反码，补码(底层，重点，难点)(对于有符号的而言)

  1. 二进制的最高位(最左位)的符号位: 0表示正数，1表示负数
  2. 正数的原码，反码，补码都一样(三码合一)
  3. 负数的反码 = 它的原码符号位不变，其它位取反
  4. 负数的补码 = 它的反码 + 1，负数的反码 = 负数的补码 - 1
  5. 0的反码，补码都是0
  6. java没有无符号数，换言之，java中的数都是有符号的
  7. 在计算机运算的时候，都是以补码的方式来运算的
  8. 当我们看到运算结果的时候，要看它的原码
  9. 补码:为了方便存储有符号位的负数，和方便加减法运算，变成只有加法的二进制运算，提高执行效率，把正数和整数统一起来了，后面有时间学学计组

5)chapter05 控制结构 Control structure

分支控制 Branch control

1.if单分支 if-else双分支 if-else if-else多分支

1.多分支可以没有else，如果所有的条件表达式都不成立，则一个执行入口都没有，多分支最多只能有一个执行入口
2.嵌套分支 Nested branches

2.switch-case-default
  1. default不需要break;
  2. 表达式数据类型，应该和case后的常量类型一致，或者是可以自动转换可以相互比较的
  3. switch表达式中表达式的返回值必须是:byte,short,int,char,enum[枚举]，String
  4. case子句中的值必须是常量，或者是常量表达式，而不能是变量
  5. default子句是可选的，当没有匹配的case时，执行default，如果没有default子句则没有任何输出
  6. case子句中如果没有break，则会按顺序执行到switch结尾
3.嵌套分支 Nested branches

如果判断的具体数值不多，而且符合byte,short,int,char,enum[],String这6种类型，建议使用switch
其他情况：对区间判断，对结果为boolean类型判断，建议使用if

循环控制 Loop control

1.for循环控制
  1. for (循环变量初始化;循环条件;循环变量迭代) { 循环体; }
  2. 循环初始值可以有多条初始化语句，但要求类型一致，并且中间用逗号隔开
  3. 循环变量迭代也可以有多条变量迭代也可以有多条变量迭代语句，中间用逗号隔开
2.while循环控制
  1. while (循环条件) {循环体;循环变量迭代}
  2. 先判断再执行
3.do-while循环控制
  1. do {循环体;循环变量迭代;} while(循环条件);
  2. 有循环四要素
  3. 先执行再判断，一定会至少执行一次
  4. 最后有一个分号;,别忘了
  5. do while是关键词
4.多重循环，嵌套循环 Nested loop
  1. 实质上，嵌套循环就是把内层循环当成外层循环的循环体，当只有内层循环的循环条件为false时，才会完全跳出内层循环，进行下一个外层循环
  2. 空心金字塔，空心菱形，化繁为简，先死后活，循序渐进，建立编程思想

成员方法 Member method

1.jvm内存方法调用机制分析 Analysis of jvm memory method calling mechanism
  1. 当程序执行到方法时，就会开辟一个独立的栈空间
  2. 当方法执行完毕，或者执行到return语句时就会返回
  3. 返回到调用方法的地方
  4. 返回后，继续执行方法后面的代码
  5. 当main方法(栈)执行完毕，整个程序退出(栈空间先进后出，后出先进)
2.成员方法带来的好处 Benefits of member methods
  1. 提高代码的复用性
  2. 可以将实现的细节封装起来，然后供其他用户程序员来调用
3.成员方法的定义 Definition of member methods
  1. 形参列表，表示成员方法的输入
  2. 返回数据类型表示成员方法输出，为void时，则不需要return返回,即没有输出
  3. 方法主体表示为了实现某一功能代码块
4.成员方法细节 Member method details
访问修饰符 Access modifier
  1. 作用是控制方法使用的范围
  2. 如果不写则为默认
  3. public protected 默认 private
返回类型 Return type
  1. 一个方法最多有一个返回值
  2. 返回类型可以为任意类型，包含基本数据类型和引用数据类型
  3. 如果方法要求有返回数据类型，则方法体中最后的执行语句必须为return 值，而且要求该值和返回值类型一致或者兼容
  4. 如果是void，则方法中可以没有return语句，或者只写return;表示方法结束
方法名 Method name

遵守驼峰命名法，最好见名知意，表示出该功能的意思即可

形参列表 Parameter list
  1. 一个方法可以没有参数，也可以有多个参数，中间用逗号隔开
  2. 参数类型可以为任意类型，包含基本数据类型和引用数据类型
  3. 调用带参数的方法时，一定对应着参数列表传入相同类型或兼容类型的参数
  4. 方法定义时的参数称为形式参数，简称形参;方法调用时的传入参数称为实际参数，简称实参，实参和形参的类型要一致或兼容，个数，顺序必须一致
方法体 Method body

里面写完成功能的具体语句，可以为输入，输出，变量，运算，分支，循环，方法调用，但里面不能再定义方法，即方法不能嵌套定义

方法调用细节说明 Method call details
  1. 同一个类中的方法可以直接调用
  2. 跨类中的方法A类调用B类的方法，需要通过对象名调用。对象名.方法名(参数)
  3. 特别说明:跨类的方法调用和方法的访问修饰符相关
方法传参机制 Method parameter(非常重要)
  1. 参数为基本数据类型，传递的是值(值拷贝),形参的任何改变不影响实参
  2. 参数为引用数据类型，传递的是地址(传递的也是值，但是值是地址),可以通过形参直接影响实参
  3. 注意:对象名只是对象的名字，它指向堆中对象的地址，不同栈中相同的对象名，不一定指向同一个地址
方法递归调用 Method recursive call
  1. 执行一个方法时，就会创建一个新的受保护的独立的栈空间
  2. 方法的局部变量(基本数据类型)是独立的，不会相互影响
  3. 如果方法中使用的是引用数据类型变量，就会共享该引用类型的数据，即同时指向同一个地址
  4. 递归必须向退出递归的条件逼近，否则就是无限递归，导致栈溢出
  5. 每一个栈都会完整的执行方法
  6. 栈是先进后出，后进先出的
5.方法重载 Method overload
  1. java中允许同一个类中，多个同名方法存在，但要求形参列表不一致
  2. 好处：减轻了起名的麻烦
  3. 要求：1.方法名相同 2.形参列表必须不同(个数，顺序，类型不同,跟形参名无关) 3.返回值无要求
6. 可变参数 Variable parameter
  1. java允许将同一个类中多个同名同功能但参数个数不同的方法，封装成一个方法
  2. 可变参数的本质就是数组，因此可以当作数组来使用
  3. 可变参数的实参可以为0个或任意个
  4. 可变参数的实参可以为数组
  5. 可变参数可以和普通类型的参数一起放在形参列表，但必须保证可变参数在最后
  6. 一个形参列表中只能有一个可变参数
  7. 方法名(int... ints)

(变量)作用域 (Variable) Scope

基本概念:

  1. 在Java编程中，主要的变量就是属性field(成员变量) 和 局部变量
  2. 我们说的局部变量一般是指在成员方法中定义的变量
  3. Java中作用域的分类: 全局变量也就是属性，作用域为整个类;局部变量，也就是除了属性之外的其他变量，作用域为定义它的代码块中(其实作用域就是大括号内)
  4. 全局变量可以不赋值，直接使用，因为有默认值，局部变量必须赋值后，才能使用，因为没有默认值

注意事项和细节:

  1. 属性和局部变量可以重名，访问时遵守就近原则
  2. 在同一个作用域中，比如在同一个成员方法中，两个局部变量，不能重名
  3. 属性的生命周期较长，伴随着对象的创建而创建，伴随着对象的销毁而销毁，局部变量，生命周期较短，伴随着它的代码块的执行而创建，伴随着代码块的结束而死亡，即在一次方法调用过程中
  4. 作用域范围不同:全局变量(属性)可以被本类使用，也可以被其他类使用(通过对象调用);局部变量只能在本类中对应的方法中使用
  5. 修饰符不同:全局变量(属性)可以加修饰符;局部变量不可以加修饰符

构造方法 Constructor

基本概念:

  1. 构造器又叫构造方法，它的方法名和类名一致
  2. 构造器的被调用时，其实对象已经存在了，它的作用是完成对象的初始化
  3. 构造器没有返回值，也不能写void

注意事项和细节:

  1. 一个类可以定义多个不同的构造器，即构造器重载
  2. 所有构造器名都必须和类名要相同
  3. 构造器是完成对象的初始化，而不是创建对象
  4. 在创建对象时，系统默认调用该类的构造方法
  5. 如果程序员没有定义构造器，系统会自动给类生成一个默认无参构造器(也叫默认构造器),可以用javap反编译看看
  6. 一旦定义了构造器，默认构造器就会被覆盖，就不能再使用默认的无参构造器了，除非再显式的定义一个无参的构造器

对象创建的流程分析 Process analysis of object creation

  1. 先加载类信息，只会加载一次
  2. 在堆中分配对象空间，就会有一个对应的地址
  3. 完成对象的初始化(三步):默认初始化(属性取数据类型的默认值)->显式初始化(属性取类中定义的值)->构造器初始化(属性取构造器传入的值)
  4. 把对象在堆中的地址，返回给对象名(对象的引用)

this 关键字 this keyword

  1. java虚拟机会给每个对象分配this，代表当前对象,即哪个对象调用，this就指向哪个对象
  2. 可以用Object的hashCode()方法来判断this和对象名指向的是不是同一个地址，hashCode()方法是根据对象在Java虚拟机的地址，返回一个整数，但它不是真正的地址
  3. this关键字可以用来访问本类的属性，方法，构造器
  4. this用于区分当前类的属性和局部变量
  5. 访问成员方法的语法，this.方法名(参数列表);
  6. 访问构造器语法:this(参数列表); 只能在构造器中使用，且必须放在第一条语句
  7. this不能在类定义外的方法中使用
  8. 如果方法中存在和属性同名的局部变量，用this可以直接访问属性的对象，不用this则就近原则，访问局部变量

匿名对象 Anonymous object

  1. new 类名()
  2. 没有名字(引用)的对象就是匿名对象，用完即销毁
