Data Structures and Algorithm

Data Structures Algorithm
發布日期:   2022-04-30

数据结构和算法的关系

  1. 数据结构(Data Structure)是一门研究组织数据方式的学科, 有了编程语言也就有了数据结构
  2. 数据结构是算法的基础, 算法(Algorithms)是程序的灵魂
  3. 程序 = 数据结构 + 算法

数据结构

线性结构 Linear structure

基本介绍

  1. 线性结构作为最常用的数据结构, 其特点是数据元素之间存在一对一的线性关系
  2. 线性结构有两种不同的存储结构, 即顺序存储结构和链式存储结构
  3. 顺序存储的线性表称为顺序表, 存储元素是连续的
  4. 链式存储的线性表称为链表, 链表中的存储元素不一定是连续的, 元素节点中存放数据元素以及相邻元素的地址信息
  5. 线性结构常见的有: 数组 队列 链表 栈

稀疏数组 Sparse array

  1. 当一个数组中大部门元素为0, 或者为同一个值的数组时, 可以使用稀疏数组来保存该数据
  2. 一般处理方式为:
    1. 记录数组一共有几行几列, 有多个不同的值
    2. 把具有不同值的元素的行列及值记录在一个小规模的数组中, 从而缩小程序的规模

队列 Queue

  1. 队列是一个有序列表, 可以用数组或者是链表来实现
  2. 遵循先入后出的原则, 即先存入队列的数据, 要先取出

链表 Linked list

  1. 链表是一个有序的列表, 以节点的方式来存储, 每个节点必须包含data域(存储数据)和next域(指向下一个节点)
  2. 链表的每个节点不一定是连续存放的
  3. 链表分为带头节点的链表和没有头节点的链表, 根据实际的需求来确定

单向链表

  1. 需要使用辅助节点来进行各种操作(遍历, 删除, 添加, 反转, 合并等)
  2. 使用栈(递归也是栈的应用) 可以反向打印单向链表的数据, 不改变链表本身的结构

双向链表

  1. pre指向前一个节点, next指向后一个节点
  2. 遍历可以向前也可以向后查找
  3. 添加(默认添加到双向链表最后)
    1. last.next = newNode;
    2. newNode.pre = last;
  4. 修改(和单向链表差不多)
  5. 删除 (自我删除)
    1. 找到要删除的节点temp
    2. temp.pre.next = temp.next;
    3. temp.next.pre = temp.pre;

单向环形链表

  1. 约瑟夫的小孩出圈问题一定要会
    1. 定义一个辅助指针helper指向, 环形链表的最后一个节点
    2. 报数到m的小孩出圈, 就是first和helper同时移动m - 1位之后, first指向的小孩出圈
  2. 相当于实现了一个队列

栈 Stack

基本介绍和应用场景

  1. 栈是一个先进后出的有序列表
  2. 栈是限制线性表中元素的插入和删除****只能在线性表的同一端进行的一种特殊线性表
    1. 允许插入和删除的一端, 为变化的一端, 称为栈顶(top)
    2. 另一端为固定的一端, 称为栈底(Bottom)
  3. 根据栈的定义可知
    1. 最先放入栈中的元素在栈底, 最后放入栈中的元素在栈顶
    2. 而删除元素刚好相反, 最后放入的元素最先删除, 最先放入的元素最后删除
  4. 栈的应用场景
    1. 子程序调用: 在跳往子程序前, 会先将下个指令的地址存到堆栈中, 直到子程序执行完后再将地址取出, 以回到原来的程序中
    2. 处理递归调用: 和子程序调用类似, 只是除了存储下一个指令的地址外, 也将参数, 区域变量等数据存入堆栈中
    3. 表达式的转换: 中缀表达式转后缀表达式与求值
    4. 二叉树的遍历
    5. 图形的深度优先(depth-first)搜索法
  5. 出栈(pop)和入栈(push)

用栈实现一个简单的计算器

  1. new一个数字栈和符号栈
  2. while循环取出表达式, 做相应判断
  3. 栈类可以封装一些方法做判断和运算

前缀表达式 中缀表达式 后缀表达式(逆波兰表达式)

中缀表达式转后缀表达式

  1. 初始化两个栈: 运算符栈s1和存储中间结果的栈s2
  2. 从左至右扫描中缀表达式
  3. 遇到数字时, 将其压入s2
  4. 遇到运算符时, 计较其与s1栈顶运算符的优先级
    1. 如果s1为空, 或栈顶运算符为左括号, 则直接将次运算符入栈
    2. 否则, 若优先级比栈顶运算符高, 也将运算符压入s1
    3. 否则, 将s1栈顶运算符弹出并压入到s2中, 再次转到(4-1)与s1中新的栈顶运算符相比较
  5. 遇到括号时:
    1. 如果是左括号, 则直接压入s1
    2. 如果是右括号, 则依次弹出s1栈顶的运算符, 并压入s2, 直到遇到左括号为止, 此时将这一对括号丢弃
  6. 重复步骤2至5, 直到表达式的最右边
  7. 将s1中剩余的运算符依次弹出并压入s2
  8. 依次弹出s2的元素并输出, 结果的逆序即为中缀表达式对应的后缀表达式

树

为什么需要树这种数据结构

  1. 数组存储, 访问元素速度快, 但做插入, 删除或整体移动时效率较低
  2. 链式存储, 插入, 删除效率较高, 但检索效率低下
  3. 树存储
    1. 能提高数据存储, 读取的效率, 比如二叉排序树
    2. 既可以保证数据的检索速度, 同时也可以保证数据的 插入, 删除, 修改的速度

树的常用术语

  1. 节点
    1. 根节点 root
    2. 父节点
    3. 子节点
    4. 叶子节点(没有子节点的节点)
  2. 节点的权(节点值)
  3. 路径(从一个节点(一般是root)找到该节点的路线)
  4. 层
  5. 子树
  6. 树的高度(最大层数)
  7. 森林(多棵子树构成森林)

二叉树

  1. 树有很多种, 每个节点最多只能有两个子节点的一种形式成为二叉树
  2. 二叉树的子节点分为左节点和右节点
  3. 满二叉树: 如果二叉树的所有叶子节点都在最后一层, 并且节点总数是 2n-1 (n为层数)
  4. 完全二叉树: 如果二叉树的所有叶子节点都在最后一层或者倒数第二层, 而且最后一层的叶子节点在左边连续, 倒数第二层的叶子节点在右边连续
  5. 遍历
    1. 前序遍历: 父节点 -> 左子树 -> 右子树
    2. 中序遍历: 左子树 -> 父节点 -> 右子树
    3. 后序遍历: 左子树 -> 右子树 -> 父节点
  6. 顺序存储二叉树
    1. 从数据存储来看, 数组的存储方式和树的存储方式可以相互转换
    2. 顺序二叉树通常只考虑完全二叉树
    3. 第n个元素(从0开始编号)的左子节点为 2 * n + 1, 右子节点为 2 * n + 2, 父节点为 (n - 1) / 2
  7. 线索化二叉树
    1. n个节点的二叉树中含有n + 1 (2n - (n - 1)) 个空指针域
    2. 利用这些空指针域, 存放指向该节点在某种遍历次序下的前驱和后继节点的指针(这种附加的指针称为线索)
    3. 因此可分为前序, 中序, 后序线索二叉树
  8. 堆
    1. 是特殊的完全二叉树, 分为大顶堆和小顶堆
    2. 每个节点的值都大于或等于其左右节点的值, 称为大顶堆
    3. 每个节点的值都小于或等于其左右节点的值, 称为小顶堆
  9. 赫夫曼树
    1. 给定n个权值作为n个叶子节点, 构造一颗二叉树, 若该树的带权路径长度(wpl)达到最小, 就是最优二叉树, 也称赫夫曼树
    2. 赫夫曼树是带权路径最短的树, 权值较大的节点离根较近
    3. 节点的权: 若将树中节点赋给一个含有着某种含义的数值, 则这个数值称为该节点的权
    4. 带权路径长度(weighted path length): 从root到该节点之间的路径长度与该节点的权的乘积
  10. 二叉排序树 Binary Sort Tree(BST)
    1. 任何一个非叶子节点, 要求其左子节点的值比当前节点的值小, 右子节点的值比当前节点的值大
    2. 如果有相同的值, 可以将该节点放在左子节点或右子节点, 尽量避免放入相同的值
    3. 如果给一个数列{1, 2, 3, 4, 5, 6}, 创建一颗二叉排序树, 就会变成一个链表, 其查询速度比单链表还慢, 所以引出平衡二叉树
  11. 平衡二叉树 AVL树
    1. 在二叉排序树的基础上实现的, 也叫平衡二叉搜索树, 以保证插叙效率较高
    2. 它是一颗空树或它的左右两个子树的高度差的绝对值不超过1, 并且左右两个子树都是一颗平衡二叉树
    3. 平衡二叉树的常用实现方法有红黑树, AVL, 替罪羊树, Treap, 伸展树
    4. 删除操作比较复杂, 其实是没有必要的, 给节点添加一个isDeleted属性, 判断是否被删除即可
  12. 二叉树的问题(引出多叉树)
    1. 需要多次进行IO操作(海量数据存在数据库或文件中)
    2. 节点海量, 也会造成二叉树的高度很大, 会降低操作速度

红黑树 red-black tree

  1. 性质
    1. 每个节点要么是黑色, 要么是红色
    2. 根节点必须是黑色
    3. 每个叶子节点(包括NIL)都是黑色
    4. 红色节点不能相连
    5. 任意节点到每个叶子节点的路径都包含数量相同的黑节点, 俗称黑高, 黑色完美平衡
    6. 如果一个节点存在黑子节点, 那么该节点肯定有两个子节点
  2. 红黑树并不是完美平衡二叉查找树, 但它是趋近于平衡状态的
  3. 红黑树的自平衡
    1. 变色
    2. 左旋
    3. 右旋
  4. 插入一定是红色插入, 因为黑色插入一定会破坏红黑树的平衡, 红色插入有可能不会破坏
    1. 红黑树为空树: 直接作为根节点, 并把节点设为黑色
    2. 插入节点的key已存在: 更新当前节点的值, 为插入节点的值
    3. 插入节点的父节点为黑节点: 直接插入即可, 无需做自平衡
    4. 插入节点的父节点为红色: 因为根节点是黑色, 如果插入节点的父节点为红节点, 那么该父节点不可能为根节点, 所以这种情况插入节点总是存在祖父节点
  5. 单独讨论插入节点的父节点为红色的情况
    1. 叔叔节点存在而且为红节点
      1. 把父亲和叔叔节点变为黑色
      2. 把爷爷节点变为红色
      3. 再将爷爷节点设置成当前节点做后续处理
    2. 叔叔节点不存在或者为黑节点, 并且插入节点的父亲节点是祖父节点的左子节点
      1. LL双红: 把父亲节点设置成黑色, 爷爷节点设置成红色, 对爷爷节点进行右旋
      2. LR双红: 对父亲节点进行左旋, 变成LL双红的情况
    3. 叔叔节点不存在或者为黑节点, 并且插入节点的父亲节点是祖父节点的右子节点
      1. RR双红: 把父亲节点设置成黑色, 爷爷节点设置成红色, 对爷爷节点进行左旋
      2. RL双红: 对父亲节点进行右旋, 变成RR双红的情况

多叉树 (多路搜索树)

  1. 允许每个节点可以有更多的数据项和子节点, 就是多叉树, 多叉树通过重新组织节点, 减少树的高度
  2. 2-3树
    1. 最简单的B树, 2-3树是由二节点和三节点构成的树
    2. 有两个子节点的节点叫二节点, 3个就是三节点, 它们要么没有节点, 要么就有2, 3个节点
  3. B树 (B-tree), B+树, B*树
    1. Balanced, 平衡的意思, 即平衡树
    2. 所有叶子节点都在同一层
    3. B树的阶: 节点的最多子节点个数
    4. B树的搜索: 从根节点开始, 二分查找, 所以搜索性能等价于在关键字全集做一个二分查找
    5. 关键字集合存放在整棵树中(非叶子节点和叶子节点)
  4. B+树
    1. B树的变体
    2. 和B树的搜索基本相同, 区别是B+树只有达到叶子节点才命中, 其搜索性能和B树一样
    3. 所有关键字都出现在叶子节点的链表中(即数据只能存在叶子节点中, 也叫稠密索引), 且链表中的关键字恰好是有序的
    4. 非叶子节点相当于叶子节点的索引(稀疏索引), 叶子节点相当于存储关键字数据的数据层
    5. 更适合文件索引系统
    6. B树和B+树各有各的应用场景
  5. B*树
    1. B+树的变体
    2. B*树定义了非叶子节点关键字个数至少为(2/3)M, 即块的最低使用率为2/3, 而B+树的块的最低使用率为B+树的1/2
    3. B*树分配新节点的概率要比B+树低, 空间使用率更高

图

为什么要有图

  1. 线性表局限于一个直接前驱和一个直接后继的关系
  2. 树也只能有一个直接前驱也就是父节点
  3. 当需要表示多对多的关系时, 我们这里就用到了图

图的常用概念

  1. 顶点(vertex)
  2. 边(edge)
  3. 路径
  4. 无向图 顶点之间的连接没有方向
  5. 有向图 顶点之间的连接有方向
  6. 带权图 边带权值的图, 这种图也叫网

图的表示方式

  1. 二维数组表示 (邻接矩阵)
    1. 表示图形中顶点之间相邻关系的矩阵
    2. 对于n个顶点的图而言, 矩阵的row和col表示的是1…n个点
    3. 需要为每个顶点都分配n个边的空间, 其实有很多边都是不存在的, 会造成空间的一定损失
  2. 链表 + 数组 (邻接表): 只关心存在的边, 不关心不存在的边, 因此没有空间浪费

图的深度优先遍历 Depth First Search (dfs)

思想
  1. 从初始访问节点(可能有多个邻接节点)出发, 首先访问第一个邻接节点, 然后再以这个被访问的邻接节点作为初始节点, 继续访问它的第一个邻接节点
  2. 简单来说, 就是每次访问完当前节点后首先访问当前节点的第一个邻接节点
  3. 这样的访问策略是优先往纵向挖掘深入, 而不是对一个节点的所有邻接节点进行横向访问
  4. 显然, 这是一个递归的过程
步骤
  1. 访问初始节点v, 并标记v为已访问
  2. 查找节点v的第一个邻接节点w
  3. 若w存在, 则继续往下执行, 若w不存在, 则回到第1步, 将从v的下一个节点继续
  4. 若w未被访问, 对w进行深度优先遍历递归(即把w当做另一个v, 然后进行步骤123)
  5. 查找节点v的邻接节点w的下一个邻接节点, 转到步骤3

图的广度优先遍历 Broad First Search (bfs)

思想

类似于一个分层搜索的过程, 广度优先遍历需要使用一个队列以保持访问过的节点的顺序, 以便按这个顺序来访问这些节点的邻接节点

步骤
  1. 访问出事节点v并标记为已访问
  2. 节点v入队列
  3. 当队列非空时, 继续执行, 否则算法结束
  4. 出队列, 取得队头节点u
  5. 查找节点u的第一个邻接节点w
  6. 若节点u的邻接节点w不存在, 则转到步骤3, 否则循环执行以下三个步骤
    1. 若节点w尚为被访问, 则访问节点w并标记为已访问
    2. 节点w入队列
    3. 查找节点u的继w邻接节点后的下一个邻接节点w, 转到步骤6

算法

复杂度

度量一个程序(算法)执行的时间有两种方法

  1. 事后统计法: 要在同一台计算机的相同状态下来运行, 才能比较哪个算法速度更快
  2. 事前估算法: 通过分析某个算法的时间复杂度来判断哪个算法更优

时间频度

  1. 概念: 一个算法中语句的执行次数, 也叫语句频度
  2. 可以忽略常数项, 低次项, 系数

时间复杂度

  1. 算法中的基本操作语句的重复执行次数是问题规模n的某个函数, 即时间频度, 用T(n)表示
  2. 算法的时间复杂度是一个函数，它定性描述该算法的运行时间, 用 O(f(n)) 表示
  3. T(n) 不同, 但时间复杂度可能相同, T(n) = O(f(n))
  4. 计算时间复杂度的方法:
    1. 用常数1代替运行时间中的所有加法常数
    2. 保留最高阶项
    3. 去除最高阶项的系数

常见的时间复杂度

  1. 常数阶 O(1) 没有循环等复杂结构
  2. 对数阶 O(logan) *a
  3. 线性阶 O(n) 循环里的n++
  4. 线性对数阶 O(nlogan) 线性阶和对数阶级合在一起
  5. 平方阶 O(n^2) 线性阶嵌套循环
  6. 立方阶 O(n^3) 线性阶嵌套循环
  7. k次方阶 O(n^k) 线性阶嵌套循环
  8. 指数阶 O(2^n)
  9. 阶乘阶 O(n!)
  10. 上面的时间复杂度是从小到大排的

空间复杂度

  1. 定义为算法锁耗费的存储空间, 它也是问题规模n的函数
  2. 做算法分析的时候, 主要讨论的时间复杂度
  3. 一些缓存产品(redis, memcache)和算法(基数排序)本质就是用空间换时间

排序算法

内部排序

  • 按将需要处理的所有数据都加载到内部存储器中进行排序

插入排序

直接插入排序

基本思想:

  1. 把n个带排序的元素看成一个有序表和一个无序表
  2. 开始时, 有序表包含一个元素, 无序表包含 n - 1 个元素
  3. 排序过程中每次从无序表中取出一个元素, 把它的排序码依次与有序表元素的排序码进行比较, 将它插入到适当位置, 成为新的有序表

问题: 当需要插入的数是较小的数时, 后移的次数明显增多, 对效率有影响, 由此引出希尔排序

希尔排序

基本思想:

  1. 把简单的插入排序改进, 成为一个更高效的版本, 也成缩小增量排序
  2. 把记录按下标的一定增量进行分组, 对每组使用直接插入排序(如果使用冒泡效率同样会很低)
  3. 随着增量逐渐减少, 每组包含的关键词越来越多
  4. 当增量减至1时, 整个文件被分成一组, 算法便终止

选择排序

基本思想: 从欲排序的数据中, 按指定的规则选出某一元素, 再依规定交换位置后达到排序的目的

简单选择排序
堆排序
  1. 利用堆这种数据结构的性质进行排序
  2. 最坏, 最好, 平均时间复杂度均为O(nlogan), 它也是不稳定排序
  3. 堆是具有以下性质的完全二叉树
    1. 每个节点的值都大于或等于其左右节点的值, 称为大顶堆
    2. 每个节点的值都小于或等于其左右节点的值, 称为小顶堆

交换排序

冒泡排序

基本思想:

  1. 通过对待排序序列从前向后, 依次比较相邻元素的值, 若发现逆序则交换
  2. 使值较大的元素逐渐从前移向后部，就像水底的气泡一样往上冒
快速排序

基本思想:

  1. 快速排序是对冒泡排序的一种改进
  2. 通过一趟排序将要排序的数据分割成独立的两部分
  3. 其中一部分的所有数据都比另外一部分的所有数据要小
  4. 然后再按此方法对这两部分数据分别进行快速排序
  5. 整个排序过程可以递归进行

归并排序

  1. 该算法采用经典的分治策略
  2. 分治法将问题分成一些小的问题然后递归求解
  3. 而治的阶段将分的阶段得到的各答案合并在一起, 即分而治之

基数排序

又叫桶排序, 属于分配式排序, 又称”桶子法”或bin sort, 顾名思义,

外部排序

  • 数据量过大, 无法全部加载到内存中, 需要借助外部存储进行排序

查找算法

顺序(线性)查找

顾名思义, 不用多说

二分查找/折半查找

  1. 非递归法
  2. 递归法

插值查找

  1. 插值查找就是在二分查找的基础上, 对 mid 索引根据要查找的 value 做一个自适应的处理
  2. 核心算法: int mid = left + (right - left) * (value - arr[left]) / (arr[right] - arr[left]);
  3. 分布均匀的有序数组可以近似的理解成数学中的等差数列, 所以插值查找一般使用在关键字分布均匀的情况

斐波那契查找

  1. 斐波那契(黄金分割法)查找原理与前两种相似, 仍然需要对有序数组进行查找, 仅仅改变 mid 的位置, mid不在是中间和插值得到
  2. 由斐波那契数列F[k] = F[k-1] + F[k-2] 的性质,可以得到 F[k]-1 = (F[k-1]-1) + (F[k-2]-1) + 1
  3. 该式说明只要顺序表的长度为F[k]-1, 则可以分成长度为 (F[k-1]-1) 和 (F[k-2]-1)的两端
  4. 所以mid = low + F(k - 1) - 1

分治算法

  1. 分而治之
  2. 经典问题
    1. 二分搜索
    2. 大整数乘法
    3. 期盼覆盖
    4. 合并算法
    5. 快速排序
    6. 线性时间选择

    8. 循环赛日程表
    9. 汉诺塔
  3. 分治法在每一层递归上都有三个步骤

    2. 解决 解决子问题
    3. 合并 合并子问题的解
