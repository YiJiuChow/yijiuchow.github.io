基本数据类型转换

1.自动类型转换(auto-convert)
  1. 当Java程序在进行赋值或者运算时，精度小的类型自动转换为精度大的数据类型
  2. 数据类型按精度(容量)大小排序

char -> int -> long -> float -> double

byte -> short -> int -> long -> float ->double

  1. 当有多种类型的数据混合运算时，系统首先自动将所有数据转换成容量最大的数据类型，然后再进行计算
  2. char，byte，short 之间不发生自动类型转换，当把具体的数值赋给它们时，先判断该数是否在其范围内,如果使用变量赋值，则判断类型
  3. char，byte，short 在计算时，首先转换成int
  4. boolean 不参与自动转换
  5. 自动提升原则：表达式结果类型自动提升为操作数中最大的类型
2.强制类型转换(force-convert)
  1. 当进行数据从大到小转换时，就需要使用强制类型转换
  2. 需要转换一串运算结果时，常用小括号提升优先级
  3. 强制类型转换会造成精度损失

原码，反码，补码(底层，重点，难点)(对于有符号的而言)

分支控制 Branch control

1.if单分支 if-else双分支 if-else if-else多分支

1.多分支可以没有else，如果所有的条件表达式都不成立，则一个执行入口都没有，多分支最多只能有一个执行入口
2.嵌套分支 Nested branches

2.switch-case-default
  1. default不需要break;
  2. 表达式数据类型，应该和case后的常量类型一致，或者是可以自动转换可以相互比较的
  3. switch表达式中表达式的返回值必须是:byte,short,int,char,enum[枚举]，String
  4. case子句中的值必须是常量，或者是常量表达式，而不能是变量
  5. default子句是可选的，当没有匹配的case时，执行default，如果没有default子句则没有任何输出
  6. case子句中如果没有break，则会按顺序执行到switch结尾
3.嵌套分支 Nested branches

如果判断的具体数值不多，而且符合byte,short,int,char,enum[],String这6种类型，建议使用switch
其他情况：对区间判断，对结果为boolean类型判断，建议使用if

循环控制 Loop control

1.for循环控制
  1. for (循环变量初始化;循环条件;循环变量迭代) { 循环体; }
  2. 循环初始值可以有多条初始化语句，但要求类型一致，并且中间用逗号隔开

  3. 先执行再判断，一定会至少执行一次
  4. 最后有一个分号;,别忘了
  5. do while是关键词
4.多重循环，嵌套循环 Nested loop
  1. 实质上，嵌套循环就是把内层循环当成外层循环的循环体，当只有内层循环的循环条件为false时，才会完全跳出内层循环，进行下一个外层循环

  1. 同一个类中的方法可以直接调用
  2. 跨类中的方法A类调用B类的方法，需要通过对象名调用。对象名.方法名(参数)
  3. 特别说明:跨类的方法调用和方法的访问修饰符相关
方法传参机制 Method parameter(非常重要)
  1. 参数为基本数据类型，传递的是值(值拷贝),形参的任何改变不影响实参
  2. 参数为引用数据类型，传递的是地址(传递的也是值，但是值是地址),可以通过形参直接影响实参
  3. 注意:对象名只是对象的名字，它指向堆中对象的地址，不同栈中相同的对象名，不一定指向同一个地址
方法递归调用 Method recursive call
  1. 执行一个方法时，就会创建一个新的受保护的独立的栈空间

  1. 子类继承了父类所有的属性和方法，非私有的属性和方法可以在子类直接访问，但是私有属性和方法不能在子类直接访问，要通过父类提供的公共方法来访问
  2. 子类必须调用父类的构造器，完成父类的初始化
  3. 当创建子类对象时，不管使用子类的哪个构造器，默认情况下总会去调用父类的无参构造器(super();可省略)，如果父类没有无参构造器，则必须在子类的构造器中用super(形参);去指定使用父类的哪个构造器完成对父类的初始化

  6. Java所有类都是Object类的子类，因此所有类都是默认继承Object类的
  7. 构造器的调用不限于父类，父类也会调用其父类的构造器，这将一直追溯到Object类(顶级父类)
  8. 子类最多只能继承一个父类(指直接继承),即Java中是单继承机制
  9. 不能滥用继承，子类和父类之间必须满足 is - a 的逻辑关系
  10. 个人总结:this();是用来调用当前类的其他构造器，super();是用来调用其父类的构造器的

  5. java Test str1 str2 str3,则args数组为{str1,str2,str3}
  6. main方法同样是静态方法，所以在main方法中可以也只能直接调用main方法所在类的静态属性和静态方法
  7. 要调用非静态方法，必须先创建该类的对象调用

代码块 Code block

基本概念
  1. 代码化块又称为初始化块，属于类中的成员[即是类的一部分]，类似于方法，将逻辑语句封装在方法啊体中，通过{}包围起来
  2. 但它和方法不同，没有方法名，没有返回，没有参数，只有方法体，而且不用通过对象或类显式调用，而是加载类时，或创建对象时隐式调用
  3. 语法: 修饰符{方法体}; 最后这个;可写也可不写
  4. 普通代码块相当于一种对构造器的补充机制，可以做初始化操作，实现代码复用
  5. 不管调用类的哪个构造器，创建对象，都会优先调用代码块的内容
注意事项
  1. static代码块也叫静态代码块，作用就是对类进行初始化，它随着类的加载而执行，并且只会执行一次，如果是普通代码块，每创建一个对象，就执行一次
  2. 类什么时候被加载? ①创建对象实例时 ②创建子类对象实例，父类也会被加载 ③使用类的静态成员(静态方法和静态属性)时
  3. 普通的代码块，在创建对象实例时，会被隐式的调用，创建一次就会调用一次，如果只是使用类的静态成员时，普通代码块不会执行
  4. 创建一个对象时，在一个类中调用顺序是(重点，难点，面试题):
    1. 调用静态代码块和静态属性初始化(注意:静态代码块和静态属性初始化调用的优先级一样，如果有多个静态代码块和多个静态变量初始化，则按定义顺序调用)
    2. 调用普通代码块和普通属性初始化(注意:普通代码块和普通属性初始化调用的优先级一样，如果有多个普通代码块和多个普通属性初始化，则按定义顺序调用)
    3. 调用构造方法
  5. 构造器的最前面其实隐含了super()和调用本类普通代码块public Test() {super();调用本类的普通代码块;...}
  6. 创建一个子类对象时，它们的静态代码块，静态属性初始化，普通代码块，普通属性初始化，构造方法的调用顺序如下(面试题):
    1. 父类的静态代码块和静态属性初始化(优先级一样，按定义顺序执行)
    2. 子类的静态代码块和静态属性初始化(优先级一样，按定义顺序执行)
    3. 父类的普通代码块和普通属性初始化(优先级一样，按定义顺序执行)
    4. 父类的构造方法
    5. 子类的普通代码块和普通属性初始化(优先级一样，按定义顺序执行)
    6. 子类的构造方法
  7. 静态代码块只能直接调用静态成员

单例设计模式 Singleton design pattern

  1. 静态方法和静态属性的经典使用
  2. 设计模式是在大量实践中总结和理论化之后优选的代码结构，编程风格，以及解决问题的思考方式，设计模式就像是经典的棋谱，不同的棋局，我们用不同的棋谱，免去我们自己再思考和摸索
饿汉式
  1. 将构造器私有化
  2. 在类的内部直接创建对象(private static)
  3. 提供一个public的static方法，返回对象
懒汉式
  1. 仍然是构造器私有化
  2. 定义一个static静态属性对象
  3. 提供一个public的static方法，在其中new一个对象再返回这个对象
  4. 只有当用户使用上一步创建的方法时，才返回对象，后面再次调用时，会返回上次创建的对象
饿汉式和懒汉式的区别
  1. 最主要区别在于创建对象的时机不同，饿汉式是在类加载就创建了对象实例，而懒汉式是在使用时才创建
  2. 饿汉式不存在线程安全问题，懒汉式存在线程安全问题
  3. 饿汉式存在浪费资源的可能，因为如果程序员一个对象都没有使用，那麽饿汉式创建的对象就浪费了，懒汉式则是在使用时才创建，就不存在这个问题
  4. 在我们JavaSE标准类中，java.lang.Runtime就是经典的单例模式(饿汉式)

final

基本概念
  1. 当不希望类被继承时，可以用final修饰
  2. 当不希望方法被子类重写时，可以用final修饰
  3. 当不希望某个属性被修改时，可以用final修饰
  4. 当不希望某个局部变量被修改时，可以用final修饰
细节及注意事项
  1. final修饰的属性一般又叫常量，一般用全大写字母和_来命名
  2. final修饰的属性在定义时，必须赋初值，且只能赋一次，以后不能修改，非静态的final属性赋值可以在三个位置:①定义时②在构造器③在代码块
  3. 如果final修饰的属性是静态的，则初始化的位置只能是:①定义时 ②在静态代码块
  4. final类不能被继承，但是可以实例化对象
  5. 如果类不是final类，但是含有final方法，则该方法虽然不能被重写，但是可以被继承
  6. 一般来说，如果一个类已经是final类，就没有必要再将方法用final修饰，因为这个类不能被继承，所以它的方法肯定也不能被重写
  7. final不能修饰构造方法
  8. final和static往往搭配使用，效率更高，不会导致类加载，底层编译器做了优化处理
  9. 包装类(Integer Double Float Boolean等都是final),String 也是final类
  10. 形参列表也可以使用final

抽象类 Abstract class

基本概念
  1. 当父类的某些方法，需要声明，但是又不确定如何实现时，可以用abstract来修饰该方法，这个方法就是抽象方法，用abstract来修饰类就是抽象类
  2. 访问修饰符 abstract 类名{}
  3. 访问修饰符 abstract 返回类型 方法名(参数列表); 没有方法体
  4. 抽象类的价值更多的是在于设计，是设计者设计好后，让子类继承并实现抽象类
  5. 抽象类是面试官爱问的知识点，在框架和设计模式使用较多
细节和注意事项
  1. 抽象类不能被实例化
  2. 抽象类不一定包含abstract方法,而且可以包含已经实现的方法
  3. 一旦类包含了abstract方法，则这个类必须声明为abstract类
  4. abstract 只能修饰类和方法
  5. 抽象类可以有任意成员(抽象类本质还是类)，
  6. 抽象类不能有方法体，即不能被实现，只要加了{}就算实现
  7. 如果一个类继承了抽象类，则它必须实现抽象类的所有抽象方法，除非它自己也声明为abstract类
  8. 抽象方法不能使用 private ,final, static 来修饰，因为这些关键字都是和重写相违背的

模板设计模式 Template design patterns

抽象类和多态的结合

接口 interface

基本概念
  1. 接口就是给出一些没有实现的方法，封装到一起，到某个类要具体使用时，再根据具体情况把这些方法写出来
  2. 语法: interface 接口名{} , class 类名 implements 接口名{}
  3. 在jdk7.0之前，接口里的所有方法都没有方法体，即都是抽象方法，在jdk8.0以后接口可以有静态方法，默认方法
  4. 接口体现了程序设计的多态和高内聚低耦合的设计思想
细节和注意事项
  1. 接口不能被实例化
  2. 接口中的所有方法都是public方法，可以不用public修饰
  3. 接口中的抽象方法，可以不用abstract修饰
  4. 一个普通类实现接口，就必须将该接口的所有方法都实现
  5. 抽象类实现接口，可以不用实现接口的方法
  6. 一个类同时可以实现多个接口,且这多个接口中可以有重名的抽象方法，这些方法视为一个方法
  7. 接口中所有的属性都是public final static的，可以省略，所以它的属性的访问形式是: 接口名.属性名
  8. 接口不能继承其它的类，但是可以继承多个别的接口 interface IC extends IA,IB{},extends是继承，implements是实现
  9. 接口的修饰符，只能是public和默认，这点和类的修饰符是一样的
实现接口和继承类的区别
  1. 接口和继承解决的问题不同
    1. 继承的价值主要在于: 解决代码的复用性和可维护性
    2. 接口的价值主要在于: 设计，设计好各种规范(方法),让其它类去实现这些方法
  2. 接口比继承更加灵活，继承是满足is - a的关系，而接口只需满足like - a的关系
  3. 接口在一定程度实现代码解耦(即 接口的规范性+动态绑定)
接口的多态特性
  1. 多态参数，和之前的同理
  2. 多态数组，和之前的同理
  3. 接口存在多态传递现象,如果 B接口 继承了 A接口，而 C类 实现了 B接口 ,那麽，实际上就相当于C类也实现了 A接口 这就是所谓的，接口多态传递现象

类的第五大成员 内部类 Inner class (重点 难点)

  1. 类的五大成员: 属性 构造器 方法 代码块 内部类
  2. 一个类的内部又完整的嵌套了另一个类结构，被嵌套的类称为内部类(inner class),嵌套其他类的类称为外部类(outer class)
  3. 内部类最大的特点就是可以直接访问私有属性，并且可以体现类与类之间的包含关系
内部类的分类
  1. 定义在外部类局部位置上(比如方法内):
    1. 局部内部类 (有类名)
    2. 匿名内部类 (没有类名,超级重点!!!)
  2. 定义在外部类的成员位置上
    1. 成员内部类 (没用static修饰)
    2. 静态内部类 (使用static修饰)
局部内部类

局部内部类定义在外部类的局部位置，比如方法中，并且有类名

  1. 局部内部类可以直接访问外部类的所有成员，包含私有的
  2. 不能添加访问修饰符，因为它的地位就是一个局部变量，局部变量是不能使用修饰符的，但是可以使用final修饰(使其不能被继承)，因为局部变量也可以使用final
  3. 作用域: 仅仅在定义它的方法或代码块中
  4. 外部类要访问局部内部类的成员，需要创建对象再访问，且必须在作用域内
  5. 局部内部类的本质仍然是一个类
  6. 外部其他类不能访问局部内部类，因为它的地位是局部变量
  7. 如果外部类和局部内部类的成员重名时，默认遵循就近原则
    1. 如果想访问外部类的成员，则可以使用外部类名.this.成员
    2. 外部类名.this本质就是外部类的对象
匿名内部类(重点) Anonymous inner class
  1. 本质还是类，没有名字，同时还是一个对象
  2. 在底层实际上会分配名字，一般是外部类名 + $ + 定义的次序(从1开始)
  3. 匿名内部类仍然是定义在外部类的局部位置
  4. 语法: new 类或接口(参数列表){类体}
  5. 匿名内部类既是一个类的定义，同时它本身也是一个对象
  6. 可以直接访问外部类的所有成员，包含私有的
  7. 不能添加访问修饰符，因为它的地位是一个局部变量
  8. 作用域:仅仅在定义它的方法或代码块中
  9. 外部其他类不能访问局部内部类，因为它的地位是局部变量
  10. 如果外部类和局部内部类的成员重名时，默认遵循就近原则
    1. 如果想访问外部类的成员，则可以使用外部类名.this.成员
    2. 外部类名.this本质就是外部类的对象
成员内部类 Member inner class
  1. 没有用static修饰，定义在外部类的成员位置
  2. 可以添加任何访问修饰符，因为它的地位就是一个外部类的成员
  3. 作用域:在整个外部类中
  4. 可以直接访问外部类的所有成员，包含私有的
  5. 外部类访问成员内部类，需要先创建成员内部类的对象
  6. 外部其他类可以访问成员内部类
    1. Outer.Inner memberInner = new Outer().new Inner();
    2. new Outer().new Inner().内部类的方法
    3. 或者在外部类中定义一个返回成员内部类对象的方法
  7. 如果外部类和局部内部类的成员重名时，默认遵循就近原则
    1. 如果想访问外部类的成员，则可以使用外部类名.this.成员
    2. 外部类名.this本质就是外部类的对象
静态内部类 Static inner class
  1. 定义在外部类的成员位置，并且有static修饰
  2. 可以添加任何访问修饰符，因为它的地位就是一个外部类的成员
  3. 作用域:在整个外部类中
  4. 可以直接访问外部类的所有静态成员，包含私有的,但不能访问非静态成员
  5. 外部类访问静态内部类，需要先创建静态内部类的对象
  6. 外部其他类可以访问成员内部类
    1. Outer.Inner staticInner = new Outer.Inner();
    2. 同样可以在外部类中定义一个返回静态内部类的普通方法，或静态方法
  7. 如果外部类和局部内部类的成员重名时，默认遵循就近原则
    1. 如果想访问外部类的成员，则可以使用外部类名.成员
    2. 因为外部类名可以直接访问其静态成员

chapter11 枚举与注解

枚举 Enumeration

自定义类实现枚举
  1. 不需要set方法,因为枚举对象值通常为只读
  2. 对枚举对象使用final static共同修饰，实现底层优化
  3. 构造器私有化
  4. 枚举对象名字母全大写,因为它是常量
  5. 对外暴露对象(使用public final static修饰)
  6. 在类的内部创建一组枚举对象
enum关键字实现枚举
  1. 使用enum关键字开发一个枚举类时，默认会继承Enum类，而且枚举类会直接变成一个final类，使用javap工具进行反编译可以看到
  2. 传统的public final static Season SPRING = new Season("春天","温暖");可以简化成SPRING("春天","温暖")，括号里的实参对应构造器形参
  3. 如果使用无参构造器创建枚举对象，则实参列表和小括号都可以省略，只需要写一个常量名
  4. 当有多个枚举对象时，使用,间隔，最后一个用分号结尾
  5. 枚举对象必须放在枚举类的行首
  6. 其他规则和自定义类实现枚举一样
enum常用方法一览表
方法名 详细描述
valueOf 将字符串转换成枚举对象，要求字符串必须为已有的常量名，否则报错(根据输入的字符串，到枚举类中查找同名的对象，找到便返回，没找到就报错)
toString 得到当前枚举常量的名称，你可以通过重写这个方法来使得到的结果更易读
equals 在枚举类型中可以直接使用”==”来比较两个枚举常量是否相等。Enum提供的这个equals方法，也是直接使用”==”实现的。它的存在是为了在Set，List和Map中使用。注意equals方法是不能重写的
hashCode Enum实现了hashCode方法和equals保持一致。它是final的，不可被重写
getDeclaringClass 得到枚举常量所属的枚举类型的Class对象。可以用它来判断两个枚举常量是否属于同一个枚举类型
name 得到当前枚举常量的名称，建议优先使用toString()
ordinal 得到当前枚举常量的次序，从0开始编号的
compareTo 枚举类型实现了Comparable接口，这样可以比较两个枚举常量的大小(该方法返回值为，当前对象的次序 - 括号内对象的次序)
clone 枚举类型不能被Clone。为了防止子类实现克隆方法，Enum实现了一个仅抛出CloneNotSupportedException异常的不变Clone()
values 使用枚举类名调用，返回枚举类[],含有其定义的所有枚举对象
enum实现枚举类注意事项
  1. 使用 enum 关键字后，就不能再继承其它类了，因为 enum 会隐式继承 Enum，而 Java 是单继承机制
  2. enum实现的枚举类，仍然还是一个类，可以实现接口

注解 Annotation

  1. 注解也被称为元数据(Metadata),用于修饰解释 包、类、方法、属性、构造器、局部变量等数据信息
  2. 和注释一样，注解不影响程序逻辑，但注解可以被编译或运行，相当于嵌入在代码中的补充信息
  3. 在 JavaSE 中，注解的使用目的比较简单，例如标记过时的功能，忽略警告等。在 JavaEE 中注解占据了更重要的角色，例如用来配置应用程序的任何切面，代替 java EE 旧版中所遗留的繁冗代码和 XML 配置等
  4. 使用 Annotation 时要在其前面增加 @ 符号, 并把该 Annotation 当成一个修饰符使用。用于修饰它支持的程序元素
  5. 三个基本的 Annotation:
    1. @Override: 限定某个方法，是重写父类方法, 该注解只能用于方法，写了注解，编译器就会检查该方法是否真的重写了父类的方法，没有重写就会报错
    2. @Deprecated: 用于表示某个程序元素(类, 方法，字段/属性，包，参数等)已过时，常用于版本升级过度
    3. @SuppressWarnings: 抑制编译器警告
  6. @Target是修饰注解的注解，称为元注解
