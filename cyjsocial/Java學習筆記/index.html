Java

一.面向对象编程 Object Oriented Programming

1.简要

面向工程是一种执行者思维，解决简单问题可以使用面向过程

面向对象是一种设计者思维，解决复杂，需要协作的问题

面向对象离不开面向过程

面向对象三大特征：继承 封装 多态

宏观上：通过面向对象进行整体设计

微观上：执行和处理数据，任然是面向过程

c面向过程 c++面向对象 java面向对象

2.Java虚拟机的内存可以分为三个区域

虚拟机栈stack 堆heap 方法区method area

1.栈的特点

每个方法被调用时都会创建一个栈帧（存储局部变量，操作数，方法出口）

栈帧先进后出，后进先出

JVM为每个线程创建一个栈。用于存放该线程执行方法的信息（实际参数，局部变量）

栈属于线程私有，不能实现线程间共享

栈是由系统自动分配，速度快，栈是一个连续的内存空间

2.堆的特点

堆用于存储创建好的对象和数组（数组也是对象）

JVM只有一个堆，被所有线程共享

不连续的内存空间，分配灵活，速度慢

在堆上的区域，会被垃圾回收器做进一步划分，例如新生代和老年代的划分

3.方法区（也是堆）的特点

方法区是Java虚拟机规范，可以有不同的实现 JDK8以后是元数据空间和堆结合起来

Jvm只有一个方法区，被所有线程共享

方法区实际也是堆，只是用于存储类，常量相关的信息

用来存放程序中永远是不变或唯一的内容（类信息[class对象，反射机制中会重点讲授],静态变量，字符串常量等）

常量池主要存放常量:如文本字符串，final常量值

4.垃圾回收机制

java的内存管理很大程度就是：堆中对象的管理

垃圾回收的两种方法；a.引用技术法 b.引用可达法

对象空间的分配：使用new创建对象

对象空间的释放：将对象赋值null即可

Minor GC 清理新生代垃圾 Major GC 清理老年代垃圾

Full GC 用于清理年轻代，年老代，永久代区域，成本较高，会对系统性能产生影响

System.gc()通知虚拟机调用Full GC 而不是直接运行

3.抽象类 abstract

  1. 有抽象方法的类只能，必须定义成抽象类
  2. 抽象类不能实例化，即不能new来实例化抽象类
  3. 抽象类可以包含属性，方法，构造方法。但是构造方法不能用来new实例，只能用来被子类调用
  4. 抽象类只能用来被继承
  5. 抽象方法必须被子类实现

4.封装 encapsulation

  1. 提高代码的安全性，复用性
  2. 高内聚 封装细节，便于修改内部代码，提高可维护性
  3. 低耦性 简化外部调用，便于使用，扩展，协作

封装的实现，使用访问控制符/访问权限修饰符
修饰符 private default默认protected public
关于protected包的两个细节

  1. 若父类和子类在同一个包中，子类可以访问父类的protected成员，也可访问父类对象的protected成员
  2. 若父类和子类不在同一包中，子类可以访问父类的protected成员，但不能访问父类对象的protected成员

5.接口 interface

  • 接口就是一个规范，是一个比抽象类更抽象的抽象类

    定义接口的详细说明

  1. 访问修饰符；只能是public或者default
  2. 接口名：和类名相同的命名机制
  3. extends：接口可以多继承
  4. 常量：接口中的属性只能是常量，总是用public static final修饰，可以省略
  5. 方法：接口中的方法只能是 public abstract 也可以省略

    要点

  6. 子类通过implements来实现接口的规范
  7. 接口不能创建对象，但是可以用于声明引用变量类型
  8. 一个类实现了接口，必须要实现接口中的所有方法，并且这些方法只能是public的
  9. jdk1.8之前（不含8），接口中只能包含静态常量和抽象方法，不能有普通属性，构造方法，普通方法
  10. jdk1.8之后（含8），接口中可以包含普通的静态方法，默认方法
  11. 接口中带方法体的方法必须为默认方法，即必须加default关键字,且默认方法可以不用实现

6.多态 polymorphism

  1. 多态和属性无关，同一个方法调用，不同对象行为完全不同
  2. 多态存在要有3个必要条件，继承，方法重写，父类引用指向子类对象
  3. 父类引用指向子类对象后，用该父类引用调用子类重写的方法，此时多态就出现了

7.关键词

1.static 静态

  1. static方法，不能调用非static成员
  2. static是属于类的，普通方法是属于对象的
  3. static属性和方法都在方法区中，而堆里只有对象，没有static属性
  4. 静态初始化块，用于类的初始化操作，初始化静态属性
  5. 在静态初始化块中不能直接访问非static成员
  6. 变量的分类和作用域:局部变量属于方法，静态变量属于类，成员变量属于对象

2.super

super 可以看作 是直接父类对象的引用，可通过super来访问父类中被子类覆盖的方法和属性
继承树的追溯

  1. 创建一个类的对象，首先会创建其父类的对象，依次追溯
  2. 调用一个类的方法时，是先在这个类中找有没有这个方法，如果有则用，没有则去查看其父类有没有这个
    方法，如果Object类里也没有就编译出错

3.final

  1. 修饰变量，被它修饰的变量不可改变，一旦被赋值，就不能被重新赋值
  2. 修饰方法，该方法不可被子类重写，但可以被重载！
  3. 修饰类，修饰的类不能被继承

4.this

  1. this()调用重载的构造方法，避免相同的初始化代码，但只能用于构造方法中，且必须位于构造方法中的第一句
  2. this不能用于static方法中
  3. this作为普通方法的隐式参数，由系统传入到方法中

8.类 class

类class 对象object
类是描述对象的抽象概念

1.属性field 静态数据

2.方法method 动态方法

3.构造器constructor 用于对象的初始化，而不是创建对象

创建对象的四步

  1. 分配对象空间，并将对象成员变量初始化为0或null
  2. 执行属性值的显式初始化
  3. 执行构造方法
  4. 返回对象的地址给相关的变量

构造器的四个要点

  1. new调用
  2. 虽然有返回值，但不能定义返回值类型，不能再构造器中使用return返回某个值
  3. 如果没有定义构造器，编译器会自动定义一个无参的构造方法，如果已经定义则不会添加
  4. 构造器的方法名必须和类名一致
    基本数据不初始化，默认为0，引用类型（例如String）不初始化，默认为null（空）

9.继承 Extends

  1. 父类也称作超类，基类
  2. java只有单继承 java中类没有多继承，接口有多继承
  3. 子类继承父类，可以得到父亲的全部属性和方法（除了父类的构造方法），但不见得可以直接访问（比如父类的私有属性和方法）
  4. 定义一个类，没有调用extends，则它的父类是java.lang.Object

10.重写 Override

其实就是覆盖父类的方法

三个必须条件

  1. “==”：方法名，形参列表相同
  2. “<=”：返回值类型和声明异常类型，子类小于
  3. “>=”：访问权限，子类大于等于父类

11.包package,导入import

  1. 包相当于文件夹对于文件的作用，用于管理类，用于解决类的重名问题
  2. 包名：域名倒着写com.oracle.test 注意com.gao和com.gao.car是两个完全独立的包
  3. import java.util.* //导入该包下所有的类，会降低编译速度，但不会影响运行速度,同名类，必须用包名.类名，才能调用
  4. 静态导入 import static ,用来导入指定类的静态属性和静态方法
